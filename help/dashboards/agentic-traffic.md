# 에이전틱 트래픽 {#agentic-traffic}

에이전틱 트래픽 대시보드는 AI 에이전트(크롤러 및 챗봇)가 사이트와 상호작용하는 방법을 보여 줍니다. 이 보기를 사용하면 총 요청 수와 일반적인 성능 관련 지표를 추적할 수 있습니다. 시장, 카테고리, 페이지, 에이전트 간 트래픽 분포도 확인할 수 있습니다. 이 대시보드에서 사용되는 데이터는 CDN 로그에서 가져온 것이므로 지표를 표시하려면 **CDN 로그 전달**&#x200B;을 구성해야 합니다. 표시된 데이터를 세분화하는 데 도움이 되는 사용자 정의 가능 필터도 있습니다.

![트래픽 분포](/help/dashboards/assets/ag-main.png)

## CDN 로그 전달 {#cdn-setup}

**CDN 로그 전달**&#x200B;이 없으면 에이전틱 트래픽 대시보드가 비어 있습니다. 에이전틱 상호작용을 보려면 **CDN 로그 전달**&#x200B;을 구성해야 합니다.  처음 로그인하면 아래 이미지에 표시된 것과 같은 메시지가 표시됩니다.

![CDN 설정](/help/dashboards/assets/ag-log-forward1.png)

**구성으로 이동**&#x200B;을 선택하면 [고객 구성 대시보드](/help/dashboards/customer-configuration.md)의 **CDN 구성** 탭으로 자동으로 이동합니다.

![온보딩 CDN 설정](/help/dashboards/assets/ag-log-forward2.png)

이 탭에서 **온보딩 CDN**&#x200B;을 선택합니다. CDN 공급자 창이 표시됩니다.

<!-- [CDN Provider](/help/dashboards/assets/ag-log-forward3.png)-->
**CDN 공급자 온보딩** 창에서:

1. CDN 공급자(예: Akamai, Adobe 관리 Fastly, Fastly, AWS Cloudfront, Azure CDN, Cloudflare 또는 기타)를 선택합니다.
2. 로그 전달을 사용하려면 **온보딩**&#x200B;을 클릭합니다.

**기타**&#x200B;를 선택하는 경우 llmo-now@adobe.com에 도움을 요청해야 합니다.

활성화되면 로그가 수집되고 대시보드는 총 에이전트 상호작용, 성공률, 시장별 히트 수, 사용자 에이전트 분석, URL 수준 성능과 같은 지표로 채워집니다.

LLM Optimizer는 CDN 로그에서 필드의 하위 집합을 처리합니다. 원시 로그 필드 이름은 CDN 공급자에 따라 다르지만 정규화되어 다음과 같이 표시됩니다.

* URL(경로만)
* 사용자 에이전트
* 상태 코드
* 레퍼러 헤더
* 호스트 헤더
* 첫 번째 바이트까지의 시간(TTFB)
* 요청 메서드
* 타임스탬프
* 콘텐츠 유형

이러한 정규화된 필드는 에이전틱 보기를 통해 노출됩니다. [참조 트래픽](/help/dashboards/referral-traffic.md) 대시보드에서 CDN 로그가 페이지 조회수 지표를 표시하는 데 사용됩니다. CDN 로그 수집 또는 이후 데이터 처리 단계에서는 개인 식별 정보(PII)가 처리되거나 저장되지 않습니다.

## 필터 {#filters}

페이지 맨 위에서 필터를 적용하여 보기를 세분화할 수 있습니다. 선택한 필터는 대시보드에 있는 **모든** 섹션에 영향을 미칩니다. 다음을 사용자 정의할 수 있습니다.

* **날짜 범위** - 표시된 데이터의 시간대를 선택합니다. 예를 들어 지난 4주를 선택합니다. **주 사용자 정의** 옵션을 선택하여 기간을 사용자 정의할 수도 있습니다.
* **카테고리** - 표시된 결과를 미리 정의된 카테고리 또는 사용자 정의 카테고리별로 필터링합니다.
* **플랫폼** - 분석할 AI 엔진을 선택합니다.
* **에이전트 유형** - 사이트와 상호작용한 AI 에이전트 유형별로 필터링합니다. 크롤러, 챗봇 또는 모든 에이전트 간에 필터링할 수 있습니다.
* **성공률** - 상호작용 품질을 기준으로 필터링합니다(높음, 중간 또는 낮음). 이 지표는 직접 성공 응답(2xx 상태 코드)과 리디렉션(3xx 상태 코드)을 모두 포함한 성공적인 HTTP 요청의 비율을 나타냅니다.
* **콘텐츠 유형** - HTML, PDF 등과 같은 다양한 콘텐츠 유형에 대한 에이전틱 상호작용을 봅니다.

원하는 필터를 선택한 후 **필터 적용**&#x200B;을 클릭하여 대시보드에 선택 사항을 적용합니다.

## 트래픽 분포 {#traffic-distribution}

트래픽 분포 보기는 에이전트 트래픽이 시장, 카테고리 및 페이지 유형에 어떻게 분산되는지 보여 줍니다. 따라서 이 보기는 AI 에이전트가 사이트와 상호작용할 때 가장 자주 액세스하는 지역, 제품 영역 또는 콘텐츠 유형을 파악하는 데 도움이 됩니다.

![트래픽 분포](/help/dashboards/assets/ag-main.png)

페이지 상단에는 주의해야 할 세 가지 주요 지표가 있습니다.

* **에이전틱 상호작용** - 이 지표는 AI 에이전트가 웹 사이트에 요청한 총 수를 나타냅니다. 여기에는 검색 엔진, 챗봇 및 기타 비인간 트래픽의 모든 트래픽이 포함됩니다.
* **성공률** - 이 지표는 직접 성공 응답과 리디렉션을 모두 포함하여 성공한 HTTP 요청의 비율을 나타냅니다.
* **평균 TTFB**- 첫 번째 바이트까지의 시간(TTFB)은 서버에서 데이터의 첫 번째 바이트가 도달하는 데 걸리는 시간을 측정합니다. 평균값은 각 코드를 반환하는 요청 수에 따라 가중치가 부여되며, 5xx 응답을 발생한 요청은 제외됩니다. 값이 낮을수록 서버 응답 시간이 빨라집니다.

각 주요 지표에 대한 트렌드 지표는 이러한 값이 이전 기간과 비교하여 시간이 지남에 따라 어떻게 변화하고 있는지를 보여 줍니다.

## 에이전틱 트래픽 트렌드 {#agentic-trends}

에이전틱 트래픽 트렌드 차트를 사용하여 주간 성공, 실패 및 전체 히트 수를 추적합니다. 따라서 시간이 지남에 따라 에이전트 활동과 성능의 변화를 모니터링할 수 있습니다. 차트 위에 마우스를 놓아 주별 기간 동안의 데이터 진행 상황을 확인할 수도 있습니다.

![에이전틱 트래픽 트렌드](/help/dashboards/assets/ag-trends.png)

## 상단 및 하단 증가 항목 {#top-bottom-movers}

상단 및 하단 증가 항목 보기는 콘텐츠에 액세스하는 AI 시스템의 방문 또는 히트 수 등 에이전틱 트래픽이 주간에 가장 많이 변화한 URL을 강조 표시합니다. **상단 증가 항목**&#x200B;은 페이지가 가시성이나 참여도를 얻고 있음을 보여 주고, **하단 증가 항목**&#x200B;은 가장 가파르게 감소한 URL을 보여 줍니다. 이를 통해 어떤 콘텐츠가 상승하고 있는지, 주의가 필요할 수 있는지, AI 기반 검색 패턴이 어디로 이동하고 있는지 빠르게 파악할 수 있습니다.

![상단 및 하단 증가 항목](/help/dashboards/assets/movers.png)

## 사용자 에이전트 및 URL 성능 분석 {#user-url-performance}

사용자 에이전트 및 URL 성능 분석 보기는 크롤러와 챗봇이 사이트와 상호작용하는 방법에 대한 추가 데이터 분류를 제공합니다. 자세한 설명은 아래 탭을 클릭하십시오.

![사용자 에이전트 및 URL 성능 분석](/help/dashboards/assets/user-agent.png)

>[!BEGINTABS]

>[!TAB 사용자 에이전트 분석]

사용자 에이전트 분석 테이블은 페이지 유형 및 에이전트 유형(예: 크롤러 대 챗봇)에 따른 트래픽 분류를 제공합니다. 이렇게 하면 어떤 AI 에이전트가 사이트의 어느 부분을 크롤링하고 있는지 쉽게 이해할 수 있습니다. 다음 카테고리가 포함되어 있습니다.

* **페이지 유형** - 페이지의 유형입니다.
* **에이전트 유형** - 크롤러 또는 챗봇 중 하나를 사용하여 페이지를 크롤링한 AI 에이전트입니다.
* **히트 수** - 해당 특정 페이지 유형에 대해 AI 에이전트가 수행한 총 요청 수입니다.

**열 구성** 버튼을 클릭하여 표시되는 지표를 사용자 정의할 수 있습니다.

>[!TAB URL 성능 분석]

URL 성능 분석 테이블에는 개별 URL에 대한 세부 보기가 표시됩니다. 여기에는 히트 수, 고유 에이전트, 최상위 에이전트, 성공률 및 카테고리가 포함됩니다. 이렇게 하면 가치가 높은 페이지를 식별하고, 크롤링 간격을 감지하며, AI 엔진에 맞게 콘텐츠를 최적화할 수 있습니다. URL은 트래픽 양에 따라 순위가 매겨집니다. 테이블에는 다음과 같은 카테고리가 포함되어 있습니다.

* **URL** - 검사한 URL입니다.
* **총 히트 수** - AI 에이전트가 해당 URL로 보낸 총 요청 수입니다.
* **고유 에이전트** - URL에 액세스한 서로 다른 AI 에이전트의 수입니다.
* **최상위 에이전트** - 해당 URL에 대해 가장 많은 트래픽을 생성한 AI 에이전트입니다.
* **최상위 에이전트 유형** - 해당 URL에 대해 가장 많은 트래픽을 생성한 AI 에이전트의 유형입니다.
* **성공률** - 직접 성공 응답과 리디렉션을 모두 포함하여 성공한 HTTP 요청의 비율입니다.
* **카테고리** - 페이지 콘텐츠와 가장 많이 일치하는 카테고리입니다.
* **평균 TTFB(ms)** - 첫 번째 바이트까지의 시간(TTFB)은 서버에서 데이터의 첫 번째 바이트가 도달하는 데 걸리는 시간을 측정합니다(밀리초). 평균값은 각 코드를 반환하는 요청 수에 따라 가중치가 부여되며, 5xx 응답을 발생한 요청은 제외됩니다. 값이 낮을수록 서버 응답 시간이 빨라집니다.
* **응답 코드** - URL에 대해 관찰되는 HTTP 상태 코드.

URL 성능 테이블에는 URL에 빠르게 액세스할 수 있는 검색 필드가 있으며, **열 구성** 버튼을 클릭하여 표시되는 지표를 사용자 정의할 수 있습니다. 각 행 끝에 있는 **세부 정보** 아이콘을 클릭하여 각 URL에 대한 추가 세부 정보를 확인할 수도 있습니다.

![URL 세부 정보](/help/dashboards/assets/details.png)

URL 세부 정보 보기는 페이지의 성능에 대한 거시적 이해를 제공하며, 페이지 성능을 인용하는 빈도, AI 응답의 감정, 해당 페이지가 언급되는 주제와 프롬프트, 시간 경과에 따른 에이전틱 및 참조 트래픽 트렌드를 보여 줍니다.

>[!ENDTABS]

두 테이블 모두에서 **내보내기** 옵션을 사용하여 table `.csv`를 다운로드하고 팀과 인사이트를 공유하거나 경영진 보고에 테이블을 포함할 수 있습니다.
